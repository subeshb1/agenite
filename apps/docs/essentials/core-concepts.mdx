## Overview

Agenite is built around several core concepts that work together to create powerful AI agents. Understanding these concepts is crucial for effectively using the framework.

## Agents

Agents are the primary building blocks in Agenite. An agent is responsible for:

### Responsibilities

1. **Message processing**: Handling user inputs and generating appropriate responses
2. **Tool orchestration**: Managing tool selection and execution
3. **State management**: Maintaining conversation context and history
4. **Execution flow**: Controlling the step-by-step execution process

### Configuration

```typescript
const agent = new Agent({
  name: string;              // Unique identifier for the agent
  provider: LLMProvider;     // Language model provider
  tools?: Tool[];           // Available tools
  instructions?: string;     // Agent behavior instructions
  description?: string;      // Agent purpose description
  agents?: Agent[];         // Sub-agents for delegation
  stateReducer?: StateReducer; // Custom state management
  initialState?: State;     // Initial agent state
  steps?: Steps;           // Custom execution steps
  middlewares?: Middleware[]; // Behavior modifiers
});
```

## Tools

Tools extend agent capabilities by providing specific functionalities. They are designed to be:

### Characteristics

- **Type-safe**: Full TypeScript support with generics
- **Validated**: Built-in JSON Schema validation
- **Error-handled**: Structured error responses
- **Flexible**: Easy to create and compose

### Structure

```typescript
const tool = new Tool<InputType>({
  name: string;           // Tool identifier
  description: string;    // Tool purpose
  version: string;        // Semantic version
  inputSchema: JSONSchema; // Input validation schema
  execute: Function;      // Tool implementation
});
```

## Providers

Providers handle communication with language models. Agenite supports:

### Supported providers

- **OpenAI**: GPT models
- **Anthropic**: Claude models
- **AWS Bedrock**: Amazon's AI models
- **Ollama**: Local models

### Common interface

All providers implement a standard interface:

```typescript
interface LLMProvider {
  generateResponse(params: {
    messages: Message[];
    stream?: boolean;
    options?: ProviderOptions;
  }): Promise<Response>;
}
```

## Execution flow

Agenite uses a step-based execution model:

### Steps

1. **LLM call**
   - Process user input
   - Determine required actions
   - Generate responses

2. **Tool call**
   - Select appropriate tools
   - Prepare tool inputs
   - Execute tools

3. **Tool result**
   - Process tool outputs
   - Handle errors
   - Update state

4. **Agent call**
   - Manage sub-agent execution
   - Handle delegation
   - Combine results

### Execution context

Each step has access to:
- Current state
- Execution history
- Token usage
- Custom context

## State management

Agenite provides flexible state management:

### Features

- Custom state types
- State reducers
- Initial state
- State persistence

### Example

```typescript
interface CustomState {
  counter: number;
  history: string[];
}

const agent = new Agent<CustomState>({
  initialState: {
    counter: 0,
    history: [],
  },
  stateReducer: {
    counter: (new, prev = 0) => prev + new,
    history: (new, prev = []) => [...prev, new],
  },
});
```

## Middleware

Middleware modifies agent behavior:

### Common uses

- Logging
- Error handling
- Context injection
- Response modification

### Structure

```typescript
const middleware = async function* (
  generator: AsyncGenerator,
  context: Context
) {
  // Pre-processing
  for await (const value of generator) {
    // Processing
    yield value;
    // Post-processing
  }
};
```

## Multi-agent systems

Agenite supports complex agent hierarchies:

### Patterns

1. **Delegation**
   - Task distribution
   - Specialized agents
   - Result aggregation

2. **Collaboration**
   - Shared context
   - Communication
   - Coordination

### Example

```typescript
const specialistAgent = new Agent({
  name: 'specialist',
  provider,
  tools: [specializedTool],
});

const coordinatorAgent = new Agent({
  name: 'coordinator',
  provider,
  agents: [specialistAgent],
  instructions: 'Coordinate tasks between specialists',
});
```
